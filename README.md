# DatabasesProject
Investigating a non-trivial, real dataset to determine its precise semantics and choosing a subset of the data that is sufficient to answer questions of interest.

Using common-sense principles to design a relational schema capable of representing the dataset, including identifying alternatives and making trade-offs.

Implementing a relational schema using the SQL Data Definition Language.

Cleaning a dataset to prepare it for importing into a postgreSQL database.

Using the SQL Data Manipulation Language to explore the dataset and answer questions about it.

Presenting technical material to a technical audience.
